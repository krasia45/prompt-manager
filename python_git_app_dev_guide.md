
# 🚀 파이썬 & Git 필수 개념 총정리 가이드 (앱 개발 기초부터 실전까지)

앱 개발(특히 데이터를 다루고 큐레이션하는 플랫폼)을 시작할 때 반드시 알아야 할 파이썬 기초 문법과 Git 용어들을 정리했습니다. 기존에 다루었던 핵심 내용에 앱의 안정성과 확장성을 높이는 필수 개념들을 추가로 보완했습니다.

---

## 🛠 Part 1. 파이썬(Python) 필수 기초 문법

### 1. 변수 (Variable) & 자료형 (Data Type)
데이터에 이름표를 붙이고(변수), 그 데이터의 형태(자료형)를 구분합니다.
*   **문자열 (`str`):** 글자 데이터. (`"EventHub"`, `"글로벌 관광객"`)
*   **정수/실수 (`int`, `float`):** 숫자 데이터. (`71`, `3.14`)
*   **불리언 (`bool`):** 참/거짓 스위치. (`True`, `False`)

```python
app_name = "EventHub"
is_prototype_ready = True
test_score = 71
```

### 2. 자료구조 (Data Structure)
여러 데이터를 효율적으로 모아서 관리하는 방법입니다. 앱 개발에서는 데이터베이스 대신 자주 쓰입니다.
*   **리스트 (`[]`):** 데이터를 순서대로 나열한 기차(서랍장).
*   **딕셔너리 (`{}`):** '키(Key): 값(Value)'의 쌍으로 묶인 명세서.

💡 **[앱 개발 핵심] 리스트와 딕셔너리의 결합 (List of Dictionaries)**
브랜드별 혜택을 랭킹이나 단순 자동 분류가 아닌, **총체적으로 큐레이션(Total Curation)**하여 보여줄 때 가장 강력한 구조입니다.
```python
curated_events = [
    {"brand": "A브랜드", "title": "봄맞이 50% 할인", "target": "내국인"},
    {"brand": "B브랜드", "title": "관광객 전용 면세 혜택", "target": "외국인 관광객"}
]
```

### 3. 제어문 (Control Flow)
프로그램의 흐름을 통제합니다.
*   **조건문 (`if / elif / else`):** 상황에 따른 분기점(선로 변환기).
```python
ai_course_score = 71
if ai_course_score >= 80:
    print("안정권입니다.")
elif ai_course_score >= 70:
    print("합격 가능성이 높은 준수한 성적입니다.")
else:
    print("재도전이 필요합니다.")
```

*   **반복문 (`for`):** 리스트 안의 데이터를 하나씩 꺼내며 작업하는 컨베이어 벨트.
*   **메인 루프 (`while True` & `break`):** 사용자가 '종료'를 누르기 전까지 앱이 꺼지지 않게 유지하는 심장.
```python
while True:
    choice = input("1. 브랜드 큐레이션 보기  2. 종료 : ")
    if choice == "2":
        print("앱을 종료합니다.")
        break # 무한 루프 탈출
```

### 4. 함수 (Function)
반복되는 코드를 부품(모듈)처럼 묶어두는 방법입니다. `def`를 사용합니다.
```python
def show_brand_curation(events):
    print("=== 브랜드별 토탈 큐레이션 ===")
    for e in events:
        print(f"[{e['brand']}] {e['title']}")
```

## 🔥 Part 2. 앱 개발을 위한 파이썬 심화 개념 (새로 추가된 내용)

### 5. 예외 처리 (Try / Except)
사용자가 숫자를 넣어야 할 곳에 글자를 넣는 등, 예상치 못한 행동을 했을 때 **앱이 튕기지(Crash) 않고 부드럽게 넘어가도록 방어**하는 필수 문법입니다. 실제 앱 서비스에서 매우 중요합니다.
```python
try:
    user_input = int(input("나이를 숫자로 입력하세요: "))
except ValueError:
    print("앗! 숫자로만 입력해주셔야 해요. 다시 입력해주세요.")
```

### 6. 모듈과 패키지 (Import)
남들이 이미 잘 만들어둔 훌륭한 코드(도구)를 내 앱으로 가져와서 쓰는 방법입니다.
```python
import json  # 데이터를 파일로 저장하기 위해 파이썬 내장 도구를 가져옴
import random # 무작위 추천 기능이 필요할 때 가져옴
```

### 7. 클래스와 객체 지향 (Class & Object)
데이터(변수)와 기능(함수)을 하나로 묶어 **'붕어빵 틀'**처럼 만드는 방법입니다. 앱이 점차 커지면 사용자를 'User 객체', 이벤트를 'Event 객체'로 찍어내어 관리하는 것이 코드를 관리하기 훨씬 수월해집니다.

## 🌿 Part 3. Git 핵심 용어 (버전 관리 및 협업)

*   **저장소 (Repository):** 내 코드와 모든 변경 기록이 담기는 마법의 폴더. (로컬/원격)
*   **커밋 (Commit):** 현재 코드의 상태를 찰칵 사진(스냅샷) 찍어 영구 기록하는 것 (세이브 포인트).
*   **브랜치 (Branch):** 원본을 건드리지 않고, 새로운 기능(예: 결제 기능)을 안전하게 개발하기 위해 파생시킨 '평행 우주'.
*   **병합 (Merge):** 브랜치에서 새 기능 개발을 마치고, 본체(main)에 합치는 작업.
*   **푸시 (Push) / 풀 (Pull):** 내 컴퓨터의 변경 사항을 원격(GitHub)으로 밀어 올리거나(Push), 원격의 최신 코드를 내 컴퓨터로 당겨오는(Pull) 것.

## 🛠 Part 4. 앱 개발 실전 Git 용어 (새로 추가된 내용)

*   **클론 (Clone):** 원격(GitHub)에 있는 프로젝트 전체를 내 컴퓨터로 처음 복사해(다운로드) 올 때 사용하는 명령어입니다.
*   **충돌 (Conflict):** 평행 우주(브랜치)에서 합칠(Merge) 때, 두 명의 개발자가 똑같은 줄의 코드를 다르게 수정했거나, 내가 이전 작업과 겹치게 작업했을 때 발생합니다. "어느 쪽 코드를 선택할래?"라고 Git이 묻는 매우 자연스러운 현상입니다.
*   **`.gitignore` (깃이그노어):** API 키값, 개인정보, 무거운 가상환경 파일 등 **GitHub에 절대 올라가면 안 되는 파일들의 목록을 적어두는 메모장**입니다. 여기에 적힌 파일은 Git이 투명인간 취급하여 업로드하지 않습니다. 보안을 위해 필수적입니다.
